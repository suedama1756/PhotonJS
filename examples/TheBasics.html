A valid binding expression must specify a target property and source expression in the form: <Target Property>:< Source Expression> (additional options will be discussed below).

Automatic Change Tracking

Although Photon supports two way binding to plain ol' JavaScript objects (POJOs), it does not currently support automatic updates using POJOs. To take advantage of Photon's automatic change tracking facilities you should inherit from the 'Model' type as shown in Person.js below. (see Modelling for more details).

Complex Expressions

Bindings are not limited to simple property accessor expressions, for example the 'Hello' message above could be expressed as {text:firstName() + ' ' + lastName()}.

Note: Model properties used within complex expressions must by default be accessed using the function call syntax, e.g. lastName() rather than lastName. This restriction can be overcome on more modern browsers (> IE8) by altering Photon's model generation options (see Modelling for more details).

Multiple Bindings

Multiple bindings can be associated with an element by wrapping them in '{}' characters.

Converters

Source values can be converted using the convertFrom and convertTo options.

Data Contexts

When working with nested object structures it can sometimes be useful to change the data source used to evaluate expressions. To do this, use the special 'data-context' property.
