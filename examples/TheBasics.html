--- layout: example title: The Basics script: TheBasics.js --- Data Binding expression in photon are expressed by default using the HTML5 compliant "data-bind" attribute. For simple bindings the attribute value should be in the form: <Target Property>:< Source Expression> (advanced options will be discussed below).
Although Photon supports two way binding to plain ol' JavaScript objects (POJOs), it does not currently support automatic updates using POJOs. To take advantage of Photon's automatic change tracking facilities you should 'inherit' from the 'Model' type (see Modelling for more details).

Binding expressions are not limited to simple property accessor expressions, for example the 'Hello' message above could be expressed as {text:firstName() + ' ' + lastName()}.

Note: Model properties used within complex expressions must by default be accessed using the function call syntax, e.g. lastName() rather than lastName. This restriction can be overcome on more modern browsers (> IE8) by altering Photon's model generation options (see Modelling for more details).
